# Moving Mole Tree

This challenge migrates Frontier-CS `algorithmic/problems/30` as a faithful `piped_stdio` interactive task. The trusted evaluator implements the original moving-mole logic from `interactor.cc`: hidden target state, subtree replies, target movement on failed queries, 500-query loop, final current-node validation, and depth-cost scoring.

## Provenance

- Source path: `algorithmic/problems/30`
- Source type: interactive, `interactor.cc`
- Agentics mode: `piped_stdio`
- Public validation: one deterministic source-shaped `n = 5000` smoke tree generated by the evaluator
- Official data: private Frontier-CS `testdata/*.in` and `testdata/*.ans` sessions packaged outside Git under `private-benchmark/`

The original interactor calls `readEof()` after a final answer, which is incompatible with running several source files against one long-lived Agentics participant process. The bundle therefore keeps the original interactor source for audit and uses a Python adapter that preserves the same hidden-state transition and scoring rules while allowing multiple source sessions to run back to back.
